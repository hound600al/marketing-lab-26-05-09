# 이도형 Brand Accelerator — Design Skill

이 스킬은 그로스 마케터 **이도형 (Lee Dohyeong)** 의 개인 포트폴리오 비주얼 아이덴티티를 사용하여 채용·자기 PR·콘텐츠 자산을 제작합니다.

## 시작하기

1. **항상** 이 폴더의 `README.md` 를 먼저 읽으세요. 콘텐츠 기본 원칙, 비주얼 토큰, 아이콘 규칙이 모두 정리되어 있습니다.
2. `colors_and_type.css` 의 CSS 토큰을 import 하거나 그대로 인라인 사용하세요. 직접 값을 적지 말고 토큰 이름을 사용하세요(`var(--blue-500)`, `var(--ink-800)` 등).
3. 시각 자산을 사용할 때는 `assets/` 폴더에서 복사해서 쓰세요 (`profile_dohyeong.jpg`, `covering_*.png` 등).
4. 슬라이드 덱은 `slides/Portfolio Deck.html` 의 구조를 참고하여 새 슬라이드를 만드세요. 1920×1080, deck-stage 컴포넌트 사용.

## 사용자가 가이드 없이 호출했을 때

사용자에게 무엇을 만들지 묻고, 다음 질문들을 던지세요:

- **무엇을 만드시려나요?** (슬라이드 덱 · 단일 페이지 · 인스타 캐러셀 · 명함 · 이력서 PDF · 기타)
- **용도는?** (특정 회사 면접용 · 일반 자기소개 · 외부 강연 · 컨택용 미니덱)
- **분량은?** (몇 슬라이드 / 몇 페이지)
- **강조하고 싶은 성과는?** (전체 6단계 그로스 · 특정 단계 · 특정 작업물)
- **추가 새 데이터가 있나요?** (없으면 기존 노션 포트폴리오 데이터만 사용)

답변을 받으면 디자인 전문가로서 다음을 모두 검토:
- 메인 카피의 톤 (1인칭 + 결과부터 + 데이터로 받침)
- 컬러 토큰의 일관된 사용 (blue-500이 강조, mint-500이 성장, rose-500이 손실)
- 헤드라인 강조의 단일 사용 (`u-blue`나 `u-blue-mark` 클래스를 한 헤드라인에 한 단어만)
- 이모지의 절제 (섹션 헤더와 6단계 인디케이터만, 본문에는 X)
- 카드 패턴의 재사용 (`.card`, `.metric` 등 시스템 클래스 활용)

## 출력 형식

작업 산출물의 종류에 따라:

- **슬라이드/시각 자료/프로토타입:** 정적 HTML 파일을 생성하고, 사용자가 바로 볼 수 있게 `done` 도구로 제시
- **프로덕션 코드 (실제 웹사이트나 앱):** `colors_and_type.css` 의 토큰을 직접 쓰지 말고 사용자 시스템에 맞게 토큰을 옮겨 적용
